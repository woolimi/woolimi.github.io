<h2 id="chapter-3-monad"><a class="heading-link" aria-hidden="true" href="#chapter-3-monad">Chapter 3: Monad</a></h2>
<h3 id="1-monad모나드와-promise"><a class="heading-link" aria-hidden="true" href="#1-monad모나드와-promise">1. Monad(모나드)와 Promise</a></h3>
<p>모나드란 함수합성을 안전하게 해주는 장치.</p>
<p>함수 f와 g를 합성한다고 할때, 수학적인 기호로는 다음과 같이 표시한다.</p>
<pre><code>f * g
</code></pre>
<p>이론적으로는 f(g(x)) 를 합성한 결과는 언제 어디서 실행하든지 항상 f(g(x))가 되어야 하지만, 현실 에서는 <strong>여러가지 외부적인 요인(메모리누수, API 서버 고장 등)으로 인해 같지 않을 수 있다</strong>.</p>
<pre><code>f(g(x)) == f(g(x)) // 이론
f(g(x)) != f(g(x)) // 현실
</code></pre>
<p>모나드는 함수합성을 안전하게 하기위한 목적을 가지고 만든 객체(Object)다. 이 객체가 map을 메소드로 가지고 있으면 functor(함수자), map과 flatMap을 메소드로 가지고 있으면 monad라고 정의하였지만 이것은 그냥 정의일 뿐. <strong>모나드가 우리에게 어떤 가치가 있는지 이해하는것이 중요</strong>하다.</p>
<h4 id="1-array"><a class="heading-link" aria-hidden="true" href="#1-array">1) Array</a></h4>
<p>Array를 일종의 모나드로 생각해볼 수 있다.</p>
<pre><code>f(g(x)) == f(g(x))     // 일반적인 경우
f(g(x)) == 실행안함      // 비어있는 배열인 경우
</code></pre>
<p>Array 모나드는 다음과 같은 목적으로 만들어진 모나드</p>
<ul>
<li>Array가 비어있다면 함수합성을 하지 않는다.</li>
<li>여러 요소의 함수합성을 가능하게 해준다.</li>
</ul>
