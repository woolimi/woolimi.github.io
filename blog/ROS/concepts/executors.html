intermediate - executors

Overview

ROS 2м—җм„ң мӢӨн–ү кҙҖлҰ¬лҠ” Executorк°Җ лӢҙлӢ№н•ңлӢӨ. ExecutorлҠ” OS мҠӨл Ҳл“ң н•ҳлӮҳ мқҙмғҒмқ„ мӮ¬мҡ©н•ҙ, л“Өм–ҙмҳӨлҠ” л©”мӢңм§ҖВ·мқҙлІӨнҠём—җ лҢҖн•ҙ кө¬лҸ…В·нғҖмқҙлЁёВ·м„ңл№„мҠӨ м„ңлІ„В·м•Ўм…ҳ м„ңлІ„ л“ұмқҳ мҪңл°ұмқ„ нҳём¶ңн•ңлӢӨ. ROS 1мқҳ spinліҙлӢӨ мӢӨн–ү м ңм–ҙк°Җ м„ёл°Җн•ң лӘ…мӢңм Ғ Executor нҒҙлһҳмҠӨк°Җ м ңкіөлҗңлӢӨ(rclcppмқҳ executor.hpp, rclpyмқҳ executors.py, rclcмқҳ executor.h). кё°ліё APIлҠ” ROS 1кіј л№„мҠ·н•ҳлӢӨ.

м•„лһҳлҠ” C++ нҒҙлқјмқҙм–ёнҠё лқјмқҙлёҢлҹ¬лҰ¬ rclcppлҘј кё°мӨҖмңјлЎң м„ӨлӘ…н•ңлӢӨ.

Basic use (кё°ліё мӮ¬мҡ©)

к°ҖмһҘ лӢЁмҲңн•ң кІҪмҡ°, л©”мқё мҠӨл Ҳл“ңм—җм„ң л…ёл“ңмқҳ л©”мӢңм§ҖВ·мқҙлІӨнҠёлҘј мІҳлҰ¬н•ҳл Өл©ҙ rclcpp::spin(node)лҘј нҳём¶ңн•ңлӢӨ.

rclcpp::init(argc, argv);
rclcpp::Node::SharedPtr node = ...
rclcpp::spin(node);  // м—¬кё°м„ң лё”лЎңнӮ№, мҪңл°ұ мІҳлҰ¬

spin(node)лҠ” лӮҙл¶Җм ҒмңјлЎң Single-Threaded ExecutorлҘј мғқм„ұВ·мӢӨн–үн•ҳлҠ” кІғкіј к°ҷлӢӨ.

rclcpp::executors::SingleThreadedExecutor executor;
executor.add_node(node);
executor.spin();

Executorмқҳ spin()мқҙ нҳём¶ңлҗҳл©ҙ, нҳ„мһ¬ мҠӨл Ҳл“ңк°Җ rclВ·лҜёл“ӨмӣЁм–ҙ кі„мёөм—җ л“Өм–ҙмҳЁ л©”мӢңм§ҖВ·мқҙлІӨнҠёлҘј л¬»кі , н•ҙлӢ№ мҪңл°ұмқ„ нҳём¶ңн•ҳлҠ” кІғмқ„ л…ёл“ңк°Җ мў…лЈҢлҗ  л•Ңк№Ңм§Җ л°ҳліөн•ңлӢӨ. лҜёл“ӨмӣЁм–ҙ QoSмҷҖ л§һм¶”кё° мң„н•ҙ, л©”мӢңм§ҖлҠ” нҒҙлқјмқҙм–ёнҠё лқјмқҙлёҢлҹ¬лҰ¬ кі„мёөм—җ нҒҗлЎң мҢ“м§Җ м•Ҡкі  лҜёл“ӨмӣЁм–ҙм—җ л‘җм—ҲлӢӨк°Җ мҪңл°ұмқҙ мІҳлҰ¬н•  л•Ң к°Җм ёмҳЁлӢӨ(ROS 1кіјмқҳ мӨ‘мҡ”н•ң м°Ёмқҙ). Wait setмқҙ вҖңнҒҗлі„лЎң н•ҳлӮҳмқҳ мқҙм§„ н”Ңлһҳк·ёвҖқ нҳ•нғңлЎң Executorм—җкІҢ вҖңл©”мӢңм§Җ мһҲмқҢвҖқмқ„ м•Ңл ӨмЈјл©°, нғҖмқҙлЁё л§ҢлЈҢ к°җм§Җм—җлҸ„ м“°мқёлӢӨ.

Single-Threaded ExecutorлҠ” м»ҙнҸ¬л„ҢнҠёлҘј мӢӨн–үн•ҳлҠ” м»Ён…Ңмқҙл„Ҳ н”„лЎңм„ёмҠӨм—җм„ңлҸ„ мӮ¬мҡ©лҗңлӢӨ(лӘ…мӢңм Ғ main м—Ҷмқҙ л…ёл“ңк°Җ мғқм„ұВ·мӢӨн–үлҗҳлҠ” кІҪмҡ°).

Types of Executors (Executor мў…лҘҳ)

rclcppлҠ” кіөнҶө л¶ҖлӘЁ нҒҙлһҳмҠӨм—җм„ң нҢҢмғқлҗң м„ё к°Җм§Җ ExecutorлҘј м ңкіөн•ңлӢӨ.

Executorм„ӨлӘ…
SingleThreadedExecutorмҠӨл Ҳл“ң н•ҳлӮҳлЎң лӘЁл“  мҪңл°ұ мІҳлҰ¬. spin(node)мқҳ мӢӨм ң кө¬нҳ„.
MultiThreadedExecutorм„Өм • к°ҖлҠҘн•ң к°ңмҲҳмқҳ мҠӨл Ҳл“ңлЎң м—¬лҹ¬ л©”мӢңм§ҖВ·мқҙлІӨнҠёлҘј лі‘л ¬ мІҳлҰ¬.
StaticSingleThreadedExecutorл…ёл“ңмқҳ кө¬лҸ…В·нғҖмқҙлЁёВ·м„ңл№„мҠӨВ·м•Ўм…ҳ кө¬мЎ°лҘј л…ёл“ңк°Җ м¶”к°Җлҗ  л•Ң н•ң лІҲл§Ң мҠӨмә”. лӮҳлЁём§Җ л‘җ ExecutorлҠ” мқҙлҹ° ліҖкІҪмқ„ мЈјкё°м ҒмңјлЎң мҠӨмә”н•ҳлҜҖлЎң, StaticмқҖ мҙҲкё°нҷ” мӢңм җм—җ лӘЁл“  кө¬лҸ…В·нғҖмқҙлЁё л“ұмқ„ л§Ңл“ңлҠ” л…ёл“ңм—җл§Ң мӮ¬мҡ©н•ҳлҠ” кІғмқҙ мўӢлӢӨ. лҹ°нғҖмһ„ л№„мҡ©мқ„ мӨ„мқёлӢӨ.

м„ё Executor лӘЁл‘җ add_node(..)лЎң м—¬лҹ¬ л…ёл“ңлҘј л„Јм–ҙ н•ң Executorк°Җ лҸҢлҰҙ мҲҳ мһҲлӢӨ. Multi-Threaded Executorм—җм„ң мӢӨм ң лі‘л ¬лҸ„лҠ” мҪңл°ұ к·ёлЈ№ м„Өм •м—җ л”°лқј лӢ¬лқјм§„лӢӨ.

Callback groups (мҪңл°ұ к·ёлЈ№)

ROS 2м—җм„ңлҠ” л…ёл“ңмқҳ мҪңл°ұмқ„ к·ёлЈ№мңјлЎң л¬¶мқ„ мҲҳ мһҲлӢӨ. rclcppм—җм„ңлҠ” Node::create_callback_group()мңјлЎң, rclpyм—җм„ңлҠ” н•ҙлӢ№ мҪңл°ұ к·ёлЈ№ нғҖмһ…мқҳ мғқм„ұмһҗлЎң л§Ңл“ лӢӨ. мқҙ мҪңл°ұ к·ёлЈ№мқҖ л…ёл“ңк°Җ мӮҙм•„ мһҲлҠ” лҸҷм•Ҳ мң м§Җлҗҳм–ҙм•ј н•ҳл©°(мҳҲ: нҒҙлһҳмҠӨ л©ӨлІ„), кө¬лҸ…В·нғҖмқҙлЁё л“ұмқ„ л§Ңл“Ө л•Ң мҳөм…ҳмңјлЎң м§Җм •н•ңлӢӨ. м§Җм •н•ҳм§Җ м•ҠмқҖ кө¬лҸ…В·нғҖмқҙлЁё л“ұмқҖ кё°ліё мҪңл°ұ к·ёлЈ№м—җ л“Өм–ҙк°„лӢӨ.

мҪңл°ұ к·ёлЈ№ нғҖмһ…(мғқм„ұ мӢң кІ°м •):

  • Mutually exclusive: мқҙ к·ёлЈ№ м•Ҳмқҳ мҪңл°ұмқҖ лҸҷмӢңм—җ мӢӨн–үлҗҳм§Җ м•ҠмқҢ.
  • Reentrant: мқҙ к·ёлЈ№ м•Ҳмқҳ мҪңл°ұмқҖ лҸҷмӢңм—җ мӢӨн–үлҗ  мҲҳ мһҲмқҢ.

м„ңлЎң лӢӨлҘё к·ёлЈ№мқҳ мҪңл°ұмқҖ н•ӯмғҒ лі‘л ¬ мӢӨн–ү к°ҖлҠҘн•ҳлӢӨ. Multi-Threaded ExecutorлҠ” мҠӨл Ҳл“ң н’Җмқ„ мҚЁм„ң мң„ к·ңм№ҷм—җ л§һкІҢ к°ҖлҠҘн•ң л§ҢнҒј мҪңл°ұмқ„ лі‘л ¬ мІҳлҰ¬н•ңлӢӨ. мҪңл°ұ к·ёлЈ№мқ„ м—¬лҹ¬ Executorм—җ лӮҳлҲ  мӨ„лӢ№ add_callback_group(..)мқ„ м“°л©ҙ, OS мҠӨмјҖмӨ„лҹ¬лЎң нҠ№м • мҪңл°ұ(мҳҲ: м ңм–ҙ лЈЁн”„мҡ© кө¬лҸ…В·нғҖмқҙлЁё)мқ„ лӢӨлҘё мҪңл°ұліҙлӢӨ мҡ°м„ мӢңн•  мҲҳ мһҲлӢӨ.

Scheduling semantics (мҠӨмјҖмӨ„л§Ғ мқҳлҜё)

мҪңл°ұ мІҳлҰ¬ мӢңк°„мқҙ л©”мӢңм§ҖВ·мқҙлІӨнҠёк°Җ лҸ„м°©н•ҳлҠ” мЈјкё°ліҙлӢӨ м§§мңјл©ҙ ExecutorлҠ” мӮ¬мӢӨмғҒ FIFO мҲңм„ңлЎң мІҳлҰ¬н•ңлӢӨ. м–ҙл–Ө мҪңл°ұмқҙ кёём–ҙм§Җл©ҙ л©”мӢңм§ҖВ·мқҙлІӨнҠёлҠ” мҠӨнғқ н•ҳлӢЁ кі„мёөмқҳ нҒҗм—җ мҢ“мқҙкі , wait setмқҖ вҖңн•ҙлӢ№ нҶ н”Ҫм—җ л©”мӢңм§Җк°Җ мһҲлҠ”м§Җ м—¬л¶ҖвҖқл§Ң Executorм—җ м•Ңл ӨмӨҖлӢӨ. ExecutorлҠ” мқҙ м •ліҙл§ҢмңјлЎң м„ңл№„мҠӨВ·м•Ўм…ҳ нҸ¬н•Ё л©”мӢңм§ҖлҘј лқјмҡҙл“ң лЎңл№ҲмңјлЎң мІҳлҰ¬н•ҳл©°, FIFO мҲңм„ңлҠ” ліҙмһҘн•ҳм§Җ м•ҠлҠ”лӢӨ.

(нғҖмқҙлЁё мқҙлІӨнҠёлҘј лӢӨлҘё л©”мӢңм§ҖліҙлӢӨ мҡ°м„ н•ҳлҚҳ лҸҷмһ‘мқҖ Eloquentм—җм„ң м ңкұ°лҗҳм—ҲлӢӨ.)

Outlook (н•ңкі„мҷҖ лҢҖм•Ҳ)

мң„ м„ё к°Җм§Җ rclcpp ExecutorлҠ” л§ҺмқҖ м• н”ҢлҰ¬мјҖмқҙм…ҳм—җ м Ғн•©н•ҳм§Җл§Ң, мӢӨмӢңк°„м„ұ(мһҳ м •мқҳлҗң мӢӨн–ү мӢңк°„, кІ°м •м„ұ, мӢӨн–ү мҲңм„ң м ңм–ҙ)мқҙ н•„мҡ”н•ң кІҪмҡ°м—җлҠ” лӢӨмқҢ к°ҷмқҖ н•ңкі„к°Җ мһҲлӢӨ.

  1. мҠӨмјҖмӨ„л§Ғ мқҳлҜёк°Җ ліөмһЎВ·нҳјн•©лҗҳм–ҙ мһҲм–ҙ м •нҳ•м Ғмқё нғҖмқҙл°Қ л¶„м„қмқҙ м–ҙл өлӢӨ.
  2. мҡ°м„ мҲңмң„ м—ӯм „: лҶ’мқҖ мҡ°м„ мҲңмң„ мҪңл°ұмқҙ лӮ®мқҖ мҡ°м„ мҲңмң„ мҪңл°ұм—җ л§үнһҗ мҲҳ мһҲлӢӨ.
  3. мҪңл°ұ мӢӨн–ү мҲңм„ңлҘј лӘ…мӢңм ҒмңјлЎң м ңм–ҙн•  мҲҳ м—ҶлӢӨ.
  4. нҠ№м • нҶ н”Ҫм—җ лҢҖн•ң нҠёлҰ¬кұ°лҘј лӮҙмһҘмңјлЎң м ңм–ҙн•  мҲҳ м—ҶлӢӨ.

Executor мһҗмІҙмқҳ CPUВ·л©”лӘЁлҰ¬ мҳӨлІ„н—Өл“ңлҸ„ л¬ҙмӢңн•  мҲҳ м—ҶлӢӨ. Static Single-Threaded Executorк°Җ мқҙлҘј нҒ¬кІҢ мӨ„мқҙм§Җл§Ң л¶ҖмЎұн•ң кІҪмҡ°к°Җ мһҲлӢӨ.

мқҙлҘј мқјл¶Җ ліҙмҷ„н•ҳлҠ” л°©н–ҘмңјлЎң:

  • rclcpp WaitSet: Executor лҢҖмӢ  кө¬лҸ…В·нғҖмқҙлЁёВ·м„ңл№„мҠӨВ·м•Ўм…ҳ л“ұм—җ м§Ғм ‘ лҢҖкё°н•  мҲҳ мһҲлӢӨ. кІ°м •м Ғмқҙкі  мӮ¬мҡ©мһҗ м •мқҳ мІҳлҰ¬ мҲңм„ңлҘј кө¬нҳ„н•  мҲҳ мһҲмңјл©°, м—¬лҹ¬ кө¬лҸ…мқҳ л©”мӢңм§ҖлҘј л¬¶м–ҙ мІҳлҰ¬н•ҳлҠ” кІғлҸ„ к°ҖлҠҘн•ҳлӢӨ. (examples_rclcpp_wait_set нҢЁнӮӨм§Җ м°ёкі .)
  • rclc Executor: micro-ROSмҡ© C нҒҙлқјмқҙм–ёнҠё лқјмқҙлёҢлҹ¬лҰ¬ rclcмқҳ Executor. мҪңл°ұ мӢӨн–ү мҲңм„ңмҷҖ нҠёлҰ¬кұ° мЎ°кұҙмқ„ м„ёл°Җн•ҳкІҢ м ңм–ҙн•  мҲҳ мһҲмңјл©°, LET(Logical Execution Time) мӢңл§ЁнӢұ м•„мқҙл””м–ҙлҘј л°ҳмҳҒн•ңлӢӨ.
