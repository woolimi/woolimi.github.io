## 개념 정리

| 개념 | 설명 |
|------|------|
| **Node** | 실행 단위. 하나의 프로세스. 퍼블리셔/서브스크라이버/서비스 서버·클라이언트 등을 포함해 **토픽·서비스·액션**으로 통신한다. |
| **Topic** | **비동기** 메시지 스트림. 퍼블리셔가 계속 보내고, 서브스크라이버가 구독한다. 1:N, N:1, N:N 가능. (예: 센서 데이터, 속도 명령) |
| **Service** | **동기** 요청–응답. 클라이언트가 한 번 요청하면 서버가 한 번 응답한다. (예: 파라미터 조회, 상태 한 번 가져오기) |
| **Action** | **비동기** 장시간 작업. 목표 전송 → 진행 피드백 수신 → 결과 수신. (예: 네비게이션, 그리퍼 제어) |
| **Interface** | Topic/Service/Action에서 주고받는 **데이터 형식** 정의. `.msg`, `.srv`, `.action` 파일로 정의된다. |

- **Topic** ↔ 메시지 타입(interface)  
- **Service** ↔ 서비스 타입(요청/응답 interface)
- **Action** ↔ 액션 타입(목표/피드백/결과 interface)

---

## 서비스 관련 명령어

```bash
ros2 service list 
# /clear
# /kill
# /reset
# /spawn
# /turtle1/set_pen
# /turtle1/teleport_absolute
# /turtle1/teleport_relative
# /turtlesim/describe_parameters
# /turtlesim/get_parameter_types
# /turtlesim/get_parameters
# /turtlesim/get_type_description
# /turtlesim/list_parameters
# /turtlesim/set_parameters
# /turtlesim/set_parameters_atomically

ros2 service type [서비스명]
ros2 service type /turtle1/teleport_absolute
# turtlesim/srv/TeleportAbsolute

ros2 interface show [service type]
ros2 interface show turtlesim/srv/TeleportAbsolute
# 
# float32 x
# float32 y
# float32 theta
# ---

ros2 service call [서비스명] [Interface] "{ ... }"
ros2 service call /turtle1/teleport_absolute turtlesim/srv/TeleportAbsolute "{x: 2, y: 2, theta: 1.57}"
```

## Topic

![ros2 topic](/ros2-topic.gif)

센서와 같은 데이터들 (계속 보내지는 데이터)를 값이 필요할때만 구독하고 해지하는 방식으로 사용할때.

```bash
ros2 topic list -t # -t: 타입까지 한번에 알려주는 옵션
# /parameter_events [rcl_interfaces/msg/ParameterEvent]
# /rosout [rcl_interfaces/msg/Log]
# /turtle1/cmd_vel [geometry_msgs/msg/Twist]
# /turtle1/color_sensor [turtlesim/msg/Color]
# /turtle1/pose [turtlesim/msg/Pose]

ros2 topic list -v # type, interface, publisher, subscriber 보여주기
# Published topics:
#  * /parameter_events [rcl_interfaces/msg/ParameterEvent] 2 publishers
#  * /rosout [rcl_interfaces/msg/Log] 2 publishers
#  * /turtle1/color_sensor [turtlesim/msg/Color] 1 publisher
#  * /turtle1/pose [turtlesim/msg/Pose] 1 publisher

# Subscribed topics:
#  * /parameter_events [rcl_interfaces/msg/ParameterEvent] 2 subscribers
#  * /turtle1/cmd_vel [geometry_msgs/msg/Twist] 1 subscriber

ros2 interface show [topic_interface]
ros2 interface show turtlesim/msg/Pose
# float32 x
# float32 y
# float32 theta
# 
# float32 linear_velocity
# float32 angular_velocity


# 토픽에서 나오는 데이터를 출력하는 명령어
ros2 topic echo [topic_type]
ros2 topic echo /turtle1/pose
```
