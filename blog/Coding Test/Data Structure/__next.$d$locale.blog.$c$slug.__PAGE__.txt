5:T19c71,<h2 id="배열-array"><a class="heading-link" aria-hidden="true" href="#배열-array">배열 (Array)</a></h2>
<p>배열은 연속된 메모리 공간에 저장된 동일한 타입의 데이터 집합이다. 인덱스를 통해 O(1) 시간에 접근할 수 있지만, 삽입/삭제 시 O(n) 시간이 걸린다.</p>
<h3 id="특징"><a class="heading-link" aria-hidden="true" href="#특징">특징</a></h3>
<ul>
<li><strong>접근</strong>: O(1) - 인덱스로 직접 접근</li>
<li><strong>검색</strong>: O(n) - 선형 검색 필요</li>
<li><strong>삽입</strong>: O(n) - 요소 이동 필요</li>
<li><strong>삭제</strong>: O(n) - 요소 이동 필요</li>
</ul>

<h2 id="링크드리스트-linked-list"><a class="heading-link" aria-hidden="true" href="#링크드리스트-linked-list">링크드리스트 (Linked List)</a></h2>
<p>링크드리스트는 노드들이 포인터로 연결된 자료구조이다. 각 노드는 데이터와 다음 노드를 가리키는 포인터를 포함한다. 삽입/삭제는 O(1)이지만 접근은 O(n)이다.</p>
<h3 id="특징-1"><a class="heading-link" aria-hidden="true" href="#특징-1">특징</a></h3>
<ul>
<li><strong>접근</strong>: O(n) - 순차 탐색 필요</li>
<li><strong>검색</strong>: O(n) - 순차 탐색 필요</li>
<li><strong>삽입</strong>: O(1) - 포인터만 변경</li>
<li><strong>삭제</strong>: O(1) - 포인터만 변경</li>
</ul>

class HashTable:
    def __init__(self, size=10):
        self.size = size
        self.table = [[] for _ in range(size)]

    def _hash(self, key):
        return hash(key) % self.size

    def insert(self, key, value):
        index = self._hash(key)
        for pair in self.table[index]:
            if pair[0] == key:
                pair[1] = value
                return
        self.table[index].append([key, value])

    def get(self, key):
        index = self._hash(key)
        for pair in self.table[index]:
            if pair[0] == key:
                return pair[1]
        return None

    def delete(self, key):
        index = self._hash(key)
        for i, pair in enumerate(self.table[index]):
            if pair[0] == key:
                del self.table[index][i]
                return

<h2 id="그래프-graph"><a class="heading-link" aria-hidden="true" href="#그래프-graph">그래프 (Graph)</a></h2>
<p>그래프는 정점(Vertex)과 간선(Edge)으로 구성된 자료구조이다. 정점들 간의 관계를 표현하는 조직도로 그래프는 정점마다 간선이 없을수도 있고 있을수도 있으며 루트 노드, 부모와 자식이라는 개념이 존재하지 않는 부분에서 트리와 다르다.</p>
<ul>
<li>정점(Vertex): 노드(node)라고도 하며, 정점에는 데이터가 저장된다.</li>
<li>간선(Edge): 노드간의 관계를 나타낸다.</li>
</ul>
<h3 id="특징-7"><a class="heading-link" aria-hidden="true" href="#특징-7">특징</a></h3>
<ul>
<li><strong>인접 리스트</strong>: 공간 O(V + E), 간선 확인 O(degree)</li>
<li><strong>인접 행렬</strong>: 공간 O(V²), 간선 확인 O(1)</li>
</ul>
