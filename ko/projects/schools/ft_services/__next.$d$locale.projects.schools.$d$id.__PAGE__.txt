f:Tc1b2,<h2 id="개요"><a class="heading-link" aria-hidden="true" href="#개요">개요</a></h2>
<p>이 프로젝트는 Kubernetes를 사용하여 다음 서비스들을 통합한 마이크로서비스 환경을 제공한다:</p>
<ul>
<li><strong>웹 서버</strong>: Nginx (HTTP/HTTPS, SSL/TLS 지원)</li>
<li><strong>데이터베이스</strong>: MySQL</li>
<li><strong>CMS</strong>: WordPress</li>
<li><strong>데이터베이스 관리</strong>: phpMyAdmin</li>
<li><strong>파일 전송</strong>: FTPS (FTP over SSL)</li>
<li><strong>모니터링 스택</strong>: InfluxDB, Grafana, Telegraf</li>
</ul>
<p>모든 서비스는 Kubernetes Pod로 배포되며, PersistentVolume을 사용하여 데이터 영속성을 보장한다.</p>

<h2 id="서비스-구성"><a class="heading-link" aria-hidden="true" href="#서비스-구성">서비스 구성</a></h2>
<h3 id="1-nginx"><a class="heading-link" aria-hidden="true" href="#1-nginx">1. Nginx</a></h3>
<ul>
<li><strong>역할</strong>: 리버스 프록시 및 웹 서버</li>
<li><strong>포트</strong>:
<ul>
<li>HTTP: 80</li>
<li>HTTPS: 443</li>
<li>SSH: 30022 (NodePort)</li>
</ul>
</li>
<li><strong>기능</strong>:
<ul>
<li>SSL/TLS 인증서 지원</li>
<li>Ingress Controller로 사용</li>
<li>SSH 서버 포함</li>
</ul>
</li>
</ul>
<h3 id="2-mysql"><a class="heading-link" aria-hidden="true" href="#2-mysql">2. MySQL</a></h3>
<ul>
<li><strong>역할</strong>: 관계형 데이터베이스</li>
<li><strong>포트</strong>: 3306 (ClusterIP)</li>
<li><strong>영속성</strong>: PersistentVolume 사용 (1Gi)</li>
<li><strong>용도</strong>: WordPress 데이터 저장</li>
</ul>
<h3 id="3-wordpress"><a class="heading-link" aria-hidden="true" href="#3-wordpress">3. WordPress</a></h3>
<ul>
<li><strong>역할</strong>: 콘텐츠 관리 시스템</li>
<li><strong>포트</strong>: 5050 (NodePort)</li>
<li><strong>영속성</strong>: PersistentVolume 사용 (1Gi)</li>
<li><strong>연결</strong>: MySQL 서비스와 연결</li>
</ul>
<h3 id="4-phpmyadmin"><a class="heading-link" aria-hidden="true" href="#4-phpmyadmin">4. phpMyAdmin</a></h3>
<ul>
<li><strong>역할</strong>: MySQL 웹 기반 관리 도구</li>
<li><strong>포트</strong>: 5000 (NodePort)</li>
<li><strong>연결</strong>: MySQL 서비스와 연결</li>
</ul>
<h3 id="5-ftps"><a class="heading-link" aria-hidden="true" href="#5-ftps">5. FTPS</a></h3>
<ul>
<li><strong>역할</strong>: SSL/TLS를 통한 파일 전송 서버</li>
<li><strong>포트</strong>:
<ul>
<li>21 (FTP 제어)</li>
<li>20 (FTP 데이터)</li>
<li>30020-30021 (Passive 모드)</li>
</ul>
</li>
<li><strong>프로토콜</strong>: vsftpd 사용</li>
</ul>
<h3 id="6-influxdb"><a class="heading-link" aria-hidden="true" href="#6-influxdb">6. InfluxDB</a></h3>
<ul>
<li><strong>역할</strong>: 시계열 데이터베이스</li>
<li><strong>포트</strong>: 8086 (ClusterIP)</li>
<li><strong>영속성</strong>: PersistentVolume 사용 (1Gi)</li>
<li><strong>용도</strong>: 모니터링 메트릭 저장</li>
</ul>
<h3 id="7-grafana"><a class="heading-link" aria-hidden="true" href="#7-grafana">7. Grafana</a></h3>
<ul>
<li><strong>역할</strong>: 모니터링 대시보드</li>
<li><strong>포트</strong>: 3000 (NodePort)</li>
<li><strong>연결</strong>: InfluxDB와 연결</li>
<li><strong>인증</strong>: 익명 접근 활성화</li>
</ul>
<h3 id="8-telegraf"><a class="heading-link" aria-hidden="true" href="#8-telegraf">8. Telegraf</a></h3>
<ul>
<li><strong>역할</strong>: 메트릭 수집 에이전트</li>
<li><strong>기능</strong>:
<ul>
<li>Kubernetes 클러스터 메트릭 수집</li>
<li>InfluxDB로 메트릭 전송</li>
<li>RBAC 권한 사용 (cluster-admin)</li>
</ul>
</li>
</ul>
<h2 id="사전-요구사항"><a class="heading-link" aria-hidden="true" href="#사전-요구사항">사전 요구사항</a></h2>
<ul>
<li><strong>Docker</strong>: 컨테이너 런타임</li>
<li><strong>Minikube</strong>: 로컬 Kubernetes 클러스터</li>
<li><strong>kubectl</strong>: Kubernetes CLI 도구</li>
<li><strong>VirtualBox</strong> (또는 다른 VM 드라이버)</li>
</ul>

<h3 id="1-kubernetes-리소스-관리"><a class="heading-link" aria-hidden="true" href="#1-kubernetes-리소스-관리">1. Kubernetes 리소스 관리</a></h3>

<li><strong>ConfigMap 생성</strong>: Telegraf 설정 파일</li>
<li><strong>리소스 배포</strong>: 모든 서비스의 Deployment, Service, PVC 등</li>

<h3 id="2-데이터-영속성"><a class="heading-link" aria-hidden="true" href="#2-데이터-영속성">2. 데이터 영속성</a></h3>

<li>WordPress: <code>/var/www/html</code></li>
<li>InfluxDB: <code>/var/lib/influxdb</code></li>
</ul>

<li><strong>Grafana</strong>: 시각화 대시보드</li>
</ul>

<h3 id="node-노드"><a class="heading-link" aria-hidden="true" href="#node-노드">Node (노드)</a></h3>
<ul>
<li>Node는 Kubernetes의 워커 머신이다.</li>
<li>Node는 VM 또는 물리적 머신일 수 있으며, 클러스터에 따라 다르다.</li>
<li>각 Node는 Pod를 실행하는 데 필요한 서비스를 가지고 있으며, 마스터 컴포넌트에 의해 관리된다.</li>
<li>Node의 서비스에는 Docker, kubelet, kube-proxy가 포함된다.</li>
</ul>
<h3 id="pod-파드"><a class="heading-link" aria-hidden="true" href="#pod-파드">Pod (파드)</a></h3>
<ul>
<li>Kubernetes는 애플리케이션 인스턴스를 호스팅하기 위해 Pod를 생성한다.</li>
<li>Pod는 고유한 클러스터 IP 주소를 가진다 (무작위로 할당됨).</li>
<li>Pod 내의 컨테이너는 IP 주소와 포트 공간을 공유하며, 항상 같은 Node에 위치하고 함께 스케줄링되며, 같은 컨텍스트에서 실행된다.</li>
</ul>
<p><strong>이 프로젝트의 Pod:</strong></p>
<ul>
<li>telegraf</li>
<li>FTPS</li>
<li>grafana</li>
<li>influxdb</li>
<li>mysql</li>
<li>nginx</li>
<li>phpmyadmin</li>
<li>wordpress</li>
</ul>
<p>총 8개의 Pod가 "default" 네임스페이스에 배포된다.</p>
<h3 id="service-서비스"><a class="heading-link" aria-hidden="true" href="#service-서비스">Service (서비스)</a></h3>
<ul>
<li>Service는 Pod의 논리적 집합과 이들에 접근하는 정책을 정의하는 추상화다.</li>
<li>Service가 타겟으로 하는 Pod 집합은 일반적으로 selector에 의해 결정된다.</li>
</ul>
<p><strong>Service 타입:</strong></p>
<ol>
<li>
<p><strong>ClusterIP</strong> (기본 설정)</p>
<ul>
<li>클러스터 내부에서만 접근 가능</li>
</ul>
</li>
<li>
<p><strong>NodePort</strong></p>
<ul>
<li>VM(Node)에 포트를 열어 외부 접근 가능</li>
</ul>
</li>
<li>
<p><strong>LoadBalancer</strong></p>
<ul>
<li>LoadBalancer로 노출된 각 서비스는 자체 IP 주소를 받는다.</li>
</ul>
</li>
</ol>
