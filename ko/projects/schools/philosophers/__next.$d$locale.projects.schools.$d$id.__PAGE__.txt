e:T6fb5,<h2 id="개요"><a class="heading-link" aria-hidden="true" href="#개요">개요</a></h2>
<p>식사하는 철학자 문제는 동시성 프로그래밍에서 자원 공유와 데드락 방지를 다루는 고전적인 문제다. 이 프로젝트는 세 가지 다른 동기화 메커니즘을 사용하여 이 문제를 해결한다:</p>
<ol>
<li><strong>philo_one</strong>: pthread와 mutex를 사용한 구현</li>
<li><strong>philo_two</strong>: pthread와 semaphore를 사용한 구현</li>
<li><strong>philo_three</strong>: 프로세스(fork)와 semaphore를 사용한 구현</li>
</ol>
<h2 id="프로젝트-구조"><a class="heading-link" aria-hidden="true" href="#프로젝트-구조">프로젝트 구조</a></h2>
<pre><code>philosophers/
├── philo_one/      # pthread + mutex 버전
├── philo_two/      # pthread + semaphore 버전
└── philo_three/    # process + semaphore 버전
</code></pre>
<p>각 디렉토리는 독립적인 프로젝트로 구성되어 있으며, 각각의 Makefile을 가지고 있다.</p>
<h2 id="각-버전별-특징"><a class="heading-link" aria-hidden="true" href="#각-버전별-특징">각 버전별 특징</a></h2>
<h3 id="philo_one"><a class="heading-link" aria-hidden="true" href="#philo_one">philo_one</a></h3>
<ul>
<li><strong>동기화 메커니즘</strong>: pthread mutex</li>
<li><strong>특징</strong>:
<ul>
<li>각 포크에 대해 개별 mutex 사용</li>
<li>홀수/짝수 철학자를 분리하여 데드락 방지</li>
<li>각 철학자마다 모니터링 스레드 생성</li>
<li><code>pickup</code>과 <code>putdown</code> mutex로 포크 집기/놓기 동작 동기화</li>
</ul>
</li>
</ul>
<h3 id="philo_two"><a class="heading-link" aria-hidden="true" href="#philo_two">philo_two</a></h3>
<ul>
<li><strong>동기화 메커니즘</strong>: pthread + semaphore</li>
<li><strong>특징</strong>:
<ul>
<li>named semaphore를 사용하여 포크 관리</li>
<li>semaphore를 통한 동시성 제어</li>
<li>각 철학자마다 개별 eating semaphore 사용</li>
<li><code>pickup</code>과 <code>putdown</code> semaphore로 포크 집기/놓기 동작 동기화</li>
</ul>
</li>
</ul>
<h3 id="philo_three"><a class="heading-link" aria-hidden="true" href="#philo_three">philo_three</a></h3>
<ul>
<li><strong>동기화 메커니즘</strong>: process (fork) + semaphore</li>
<li><strong>특징</strong>:
<ul>
<li>각 철학자가 별도의 프로세스로 실행</li>
<li>프로세스 간 통신을 위해 named semaphore 사용</li>
<li>부모 프로세스가 자식 프로세스들을 관리</li>
<li>철학자가 죽으면 모든 프로세스를 종료</li>
</ul>
</li>
</ul>

<h3 id="매개변수-설명"><a class="heading-link" aria-hidden="true" href="#매개변수-설명">매개변수 설명</a></h3>
<ol>
<li><strong>number_of_philosophers</strong>: 철학자의 수 (최소 2명)</li>
<li><strong>time_to_die</strong>: 밀리초 단위. 철학자가 마지막 식사 후 이 시간 동안 식사를 하지 않으면 죽는다</li>

<li><strong>number_of_times_each_philosopher_must_eat</strong> (선택): 각 철학자가 최소한 먹어야 하는 식사 횟수. 이 매개변수가 제공되면 모든 철학자가 이 횟수만큼 식사를 완료하면 시뮬레이션이 종료된다.</li>
</ol>

<h3 id="출력-형식"><a class="heading-link" aria-hidden="true" href="#출력-형식">출력 형식</a></h3>
<p>프로그램은 다음과 같은 형식으로 상태를 출력한다:</p>
<pre><code>[timestamp_in_ms] [philosopher_number] [status]
</code></pre>
<p>상태 종류:</p>
<ul>
<li><code>is thinking</code>: 철학자가 생각 중</li>

<li><code>is eating</code>: 식사 중</li>
<li><code>is sleeping</code>: 잠자는 중</li>
<li><code>died</code>: 죽음</li>
</ul>
<p>예시:</p>
<pre><code>0 1 is thinking
0 2 is thinking
200 1 has taken a fork
200 1 has taken a fork
200 1 is eating
400 1 is sleeping
</code></pre>
