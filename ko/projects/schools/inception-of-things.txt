1a:T9249,<h2 id="개요"><a class="heading-link" aria-hidden="true" href="#개요">개요</a></h2>
<p>Kubernetes 학습을 위한 실습 프로젝트다. k3s, k3d, ArgoCD를 활용하여 Kubernetes 클러스터 구축 및 애플리케이션 배포를 실습한다.</p>
<h2 id="프로젝트-구조"><a class="heading-link" aria-hidden="true" href="#프로젝트-구조">프로젝트 구조</a></h2>
<p>이 프로젝트는 세 가지 주요 실습으로 구성되어 있다:</p>
<ul>
<li><strong>P1</strong>: k3s 클러스터 구축 (마스터 노드 + 워커 노드)</li>
<li><strong>P2</strong>: Kubernetes Ingress를 활용한 멀티 애플리케이션 배포</li>
<li><strong>P3</strong>: k3d와 ArgoCD를 사용한 GitOps 기반 CI/CD 파이프라인</li>
</ul>
<h2 id="사전-요구사항"><a class="heading-link" aria-hidden="true" href="#사전-요구사항">사전 요구사항</a></h2>

<li>VirtualBox</li>
</ul>

<h2 id="기술-스택-설명"><a class="heading-link" aria-hidden="true" href="#기술-스택-설명">기술 스택 설명</a></h2>
<h3 id="k3s"><a class="heading-link" aria-hidden="true" href="#k3s">k3s</a></h3>
<p>k3s는 엣지 컴퓨팅, IoT, 리소스 제약 환경을 위해 설계된 경량의 프로덕션급 Kubernetes 배포판이다. 의존성을 줄이고 바이너리 크기를 작게 하며 클러스터 설정을 단순화한다. 빠른 배포와 저사양 장치에 이상적이다.</p>
<p>Kubernetes는 K8s로 표기되는 10글자 단어다. 따라서 Kubernetes의 절반 크기인 것을 K3s로 표기한다.</p>
<h3 id="k3d"><a class="heading-link" aria-hidden="true" href="#k3d">k3d</a></h3>
<p>k3d는 Docker에서 k3s(Rancher Lab의 최소 Kubernetes 배포판)를 실행하기 위한 경량 래퍼다. k3d를 사용하면 Docker에서 단일 및 다중 노드 k3s 클러스터를 쉽게 생성할 수 있어 Kubernetes에서 로컬 개발에 적합하다.</p>
<h3 id="argocd"><a class="heading-link" aria-hidden="true" href="#argocd">ArgoCD</a></h3>
<p>Kubernetes용 지속적 배포(CD) 도구다. Git 저장소에 정의된 원하는 상태와 Kubernetes 애플리케이션이 동기화되도록 보장한다.</p>
<h3 id="namespace-vs-pod"><a class="heading-link" aria-hidden="true" href="#namespace-vs-pod">Namespace vs Pod</a></h3>
<p><strong>Namespace</strong>는 Kubernetes 내의 가상 클러스터로, 리소스를 격리하여 여러 팀이나 애플리케이션이 공존할 수 있도록 한다. <strong>Pod</strong>는 Kubernetes에서 배포 가능한 가장 작은 단위로, 스토리지와 네트워킹을 공유하는 하나 이상의 컨테이너를 포함한다.</p>
<p>Namespace는 Pod를 논리적으로 그룹화하지만, 실제 워크로드는 Pod에서 실행된다.</p>
